Modeling a lightcurve with a design matrix and Gaussian Process regression¶

A typical Kepler/K2 lightcurve exhibits secular trends–changes to the precision flux times series over timescales comparable to the observation season. Secular trends arise from either astrophysical processes—such as starspot evolution—or more mundane instrumental artifacts. The trends are difficult to model, even if the signal’s origin can be uniquely identified. Flux leaks from pre-defined apertures from a growing-or-shinking point spread functions (PSFs). A geenerative model of the secular evolution of the derived aperture photometry would require precision knowledge of the PSF 2D flux distribution, its wavelength dependence, and its time variability, all specified at a precision adequate to reproduce parts per million photometry. Such a model is rarely available or precise enough, and so “data driven” alternatives have prevailed. Gaussian process regression has been one of the most popular and most successful.

In this tutorial we demonstrate some techniques for modeling lightcurves with statistical techniques that incorporate Gaussian processes to model the secular trends.

First we will retrieve the target pixel file (TPF) data from MAST, and generate our own aperture photometry lightcurve. We will pick an extreme target: a heavily spotted “Weak Lined T-Tauri Star” (WTTS), which exhibits a massive peak-to-valley sinusoidal amplitude.

Let’s manually fit a sine wave to this data. Remarkably, a sine wave can be fit with linear least squares if we write the problem as a sum of sines and cosines. Writing the problem this way avoids having to fit the non-linear parameter phase. Instead we analytically determine the coefficients of the terms in front of our design matrix elements, which also includes a quadratic polynomial:

def design_matrix(x, n_poly=3):
    '''assemble the design matrix based on the equation above'''
    sin_vector = np.sin(2.0*np.pi*x/P_fit)
    cos_vector = np.cos(2.0*np.pi*x/P_fit)

    A = np.concatenate((np.expand_dims(sin_vector, 1),
                        np.expand_dims(cos_vector, 1),
                        np.vander(x, n_poly)), axis=1)
    return A

def design_matrix_harmonics(x, n_poly=3, n_harmonics=3):
    '''assemble the design matrix based on the equation above'''
    k_vectors = np.arange(1, n_harmonics+1, 1)[:, np.newaxis]
    sin_vectors = np.sin(2.0*np.pi*x*k_vectors/P_fit)
    cos_vectors = np.cos(2.0*np.pi*x*k_vectors/P_fit)
    print(sin_vectors.shape)
    A = np.concatenate((sin_vectors.T,
                        cos_vectors.T,
                        np.vander(x, n_poly)), axis=1)
    return A

def linear_regression_iid(A, y, yerr):
    '''compute linear regression assuming white noise'''
    ATA = np.dot(A.T, A / yerr[:, None]**2)
    sigma_w = np.linalg.inv(ATA)
    mean_w = np.linalg.solve(ATA, np.dot(A.T, y/yerr**2))
    yfit = np.matmul(mean_w, A.T)
    return yfit

Notice that the residuals are correlated. They do not look like white noise.
Let’s simulate the correlated structure using a Gaussian process. We will construct a “covariance matrix” which describes to-what-extent adjacent points know about each other.
In [18]:

def make_covariance_matrix(x, yerr, amp=0.5, ell=1.3):
    '''constructs the covariance matrix
    params:
        - amplitude
        (units of variance, i.e. flux sqaured)
        - length scale
        (units of time)
    '''
    covariance = amp * np.exp(-0.5 * (x[:, None]-x[None, :])**2 / ell**2) + np.diag(yerr ** 2)
    return covariance

def linear_regression_cov(A, y, covariance_matrix):
    '''compute linear regression assuming white noise'''
    factor, flag = cho_factor(covariance_matrix)
    S_inv = np.dot(A.T, cho_solve((factor, flag), A))
    S = np.linalg.inv(S_inv) # This inverse is cheap since it's ~ 5 x 5
    mean_w_new = np.dot(S,  np.dot(A.T, cho_solve((factor, flag), y)) )
    yfit = np.matmul(mean_w_new, A.T)
    return yfit

def lin_regress(A, y, yerr, return_dict=False):
    ATA = np.dot(A.T, A / yerr[:, None]**2)
    sigma_w = np.linalg.inv(ATA)
    mean_w = np.linalg.solve(ATA, np.dot(A.T, y/yerr**2))
    if return_dict:
        yfit = np.matmul(mean_w, A.T)
        resid = y - yfit
        chisq = np.sum(resid**2)
        N_data, k_params = A.shape
        AIC = chisq + 2.0*k_params + (2.0*k_params*(k_params+1))/(N_data-k_params-1)
        BIC = chisq + k_params * np.log(N_data)
        out = {'yfit':yfit,
               'resid':resid,
               'chisq':chisq,
               'AIC':AIC,
               'BIC':BIC}
        return out
    else:
        return mean_w
